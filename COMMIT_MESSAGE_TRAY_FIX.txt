fix: System tray icon not appearing in packaged Windows builds

## Problem
System tray icon was not appearing in packaged Windows builds (installer and portable), making it impossible for users to access the quit functionality.

### Root Cause
Icon files were not being included in the packaged application. The `buildResources` configuration in electron-builder only provides icons for the installer/executable, not for runtime use by the application itself.

**What was happening:**
- Dev mode: Icons loaded from `assets/icons/` ✅
- Packaged: Icons didn't exist at runtime ❌
- Result: `nativeImage.createFromPath()` failed silently
- Tray creation failed → no tray icon → no quit option

## Solution

### 1. Added extraResources Configuration
Added `extraResources` to package.json to copy icon files to the resources directory at build time:

```json
"extraResources": [
  {
    "from": "assets/icons/",
    "to": "icons/"
  }
],
```

**Effect:**
- Icons now copied to: `resources/icons/` (next to app.asar)
- Available at runtime via: `process.resourcesPath/icons/`
- All 7 icon files included (16, 32, 48, 64, 128, 256, 512)

### 2. Updated Icon Path Resolution
Fixed icon path in main.js `createTray()` function:

**Before:**
```javascript
iconPath = path.join(process.resourcesPath, 'app.asar', 'assets', 'icons', 'icon-32.png');
// ❌ This path never existed - icons weren't in asar
```

**After:**
```javascript
iconPath = path.join(process.resourcesPath, 'icons', 'icon-32.png');
// ✅ Correctly points to extraResources location
```

### 3. Enhanced Debugging
Added comprehensive logging and file existence checks:
- Log icon path being used
- Check if file exists before creating tray
- Log success/failure clearly
- Helps diagnose issues in production

## Notes

### electron-builder Asset Types
- **buildResources:** Build-time only (installer icons)
- **extraResources:** Runtime assets (copied to resources/)
- **files:** App code (packaged in app.asar)

System tray icons must use **extraResources** to be available at runtime.
